**変更内容の詳細**:

#### 仕様ファイル (spec.md)

**ユーザーストーリ定義**:

1. **US1: 社員情報を新規登録する (P1)**
   - POST /employees エンドポイント
   - name, email, department を受け付ける
   - id は自動採番、created_at/updated_at は自動設定
   - 201 Created で登録データを返す
   - 400 Bad Request (検証エラー)、409 Conflict (メール重複) をサポート

2. **US2: 社員一覧と個別情報を取得する (P1)**
   - GET /employees: 全社員一覧を返す (200 OK)
   - GET /employees/{id}: 特定社員の詳細を返す (200 OK, 404 Not Found)

3. **US3: 社員情報を更新する (P2)**
   - PUT /employees/{id}: 全項目更新 (200 OK, 404 Not Found)
   - PATCH /employees/{id}: 部分更新 (200 OK, 404 Not Found)

4. **US4: 社員情報を削除する (P3)**
   - DELETE /employees/{id}: 削除実行 (204 No Content, 404 Not Found)

**機能要件** (FR-001 ～ FR-009):

- CRUD エンドポイント実装
- メールアドレス一意性制約
- 標準 HTTP ステータスコード
